# Root Voca 진도 누적 점검 및 수정 v2.5.2

## 확인된 원인

기존 v2.5.1은 로그인 후 Firestore 데이터를 무조건 우선 적용했습니다. DAY 완료 직후에는 로컬 저장은 즉시 수행되지만 클라우드 저장은 1.2초 뒤에 실행되므로, 학습 종료 직후 창을 닫거나 네트워크가 지연되면 다음 접속 때 오래된 클라우드 상태가 최신 로컬 진도를 덮을 수 있었습니다.

그 결과 전날 DAY 1을 완료했더라도 다음 날 다시 DAY 1로 표시될 수 있었습니다.

## v2.5.2 수정사항

1. 로컬과 클라우드 상태에 갱신시각을 함께 기록합니다.
2. 구버전 데이터가 섞여 있으면 더 많이 진행된 상태를 우선합니다.
3. 완료 DAY 이력은 로컬·클라우드 양쪽 값을 합쳐 후퇴하지 않게 합니다.
4. DAY 완료 시 클라우드에 즉시 저장합니다.
5. 탭이 숨겨지거나 창이 닫힐 때 마지막 저장을 한 번 더 시도합니다.
6. 완료 이력이 누락되어도 해당 DAY의 모든 단어를 실제로 학습한 기록이 있으면 순차적으로 완료 DAY를 복구합니다.
7. 복구가 발생하면 “기존 학습기록을 복구했습니다. DAY N부터 이어서 진행합니다.”라고 안내합니다.
8. 관리자 DAY 변경도 로컬과 클라우드에 같은 갱신시각으로 저장합니다.

## 자동 시뮬레이션 결과

- 로컬 DAY 1 완료 + 오래된 클라우드 DAY 1 미완료: DAY 2 유지 — PASS
- 더 최신 클라우드 DAY 2 완료: DAY 3 표시 — PASS
- completedDays 누락 + DAY 1 전체 단어 학습기록 존재: DAY 2 자동복구 — PASS
- DAY 완료 즉시 클라우드 저장 호출 — PASS
- JavaScript 전체 구문검사 — PASS

## 적용 후 확인

학습자 8081로 로그인했을 때 다음 중 하나가 표시되어야 합니다.

- 기존 기록 복구 성공: `기존 학습기록을 복구했습니다. DAY 2부터 이어서 진행합니다.`
- 이미 완료기록이 정상인 경우: 홈 화면에 `오늘 반드시 해야 할 학습은 DAY 2`

여전히 DAY 1이면 관리자 2007로 로그인하여 관리 화면에서 학습자의 오늘 DAY를 2로 한 번 지정합니다. 이 조치는 DAY 1을 완료로 고정하며 이후부터는 자동 누적됩니다.
