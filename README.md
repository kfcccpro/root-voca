능률VOCA 어원편의 기존 어원·단어 순서를 보존하면서, 실제 학습량을 기준으로 60DAY를 18DAY 집중과정으로 재배치한 단일 학습자용 웹앱입니다.

- 최종 버전: `2026.07.24 FINAL`
- 진도 저장 스키마: `V3`
- 기본 완료 보고 이메일: `sk01197375068@gmail.com`
- 다른 앱과의 병합: **미실시**

## 최종 데이터 현황

| 항목 | 결과 |
|---|---:|
| ROOT·접두사·접미사 학습 단위 | 360개 |
| 표제어 | 1,410개 |

원자료에 명시적 분해식이나 화살표 풀이가 없는 단어에는 임의의 어원 설명을 만들어 넣지 않았습니다. 해당 항목은 앱에서 ROOT와 표제어를 연결하는 안전한 보조 표시를 사용합니다.

## 학습 기능

- 오늘의 DAY만 명확하게 활성화
- 관리자 설정 시작·종료시간 및 오늘 분량 표시
- 남은 시간·남은 단어 원형 게이지
- 약 15분 단위의 4개 집중블록
- ROOT 의미와 어원 구성 순차 나타나기
- 태블릿 4지선다 우선
- 오답 철자만 직접 타이핑
- 숙달 항목의 설명 단계 단축
- 블록 완료 즉시 보상
- DAY 완료 후 다음 DAY 자동 진급
- DAY 18 완료 후 과정 재시작 차단

## 진도·학습 기록

다음 항목을 브라우저 `localStorage`에 저장합니다.

- 현재 DAY와 블록
- ROOT·단어 진행 위치
- 정답·오답 결과
- 오답 복습 큐와 철자 입력 횟수
- 단어별 숙달도
- 실제 학습시간
- DAY 완료와 관리자 강제 완료 이력
- 메일 발송·재시도 상태

관리자 화면의 **진도 백업**과 **진도 복원**으로 JSON 백업 파일을 내려받거나 복원할 수 있습니다. V2 및 이전 진도 형식은 V3로 자동 변환합니다.

> 현재 최종본은 단일 기기 운영 기준입니다. 태블릿과 PC 사이에서 자동으로 같은 기록을 사용하려면 추후 통합 단계에서 온라인 DB를 연결해야 합니다.

## 완료 메일

DAY 완료 보고서는 기본적으로 `sk01197375068@gmail.com`으로 발송하도록 설정되어 있습니다.

실제 발송에는 Netlify 환경변수가 필요합니다.

- `RESEND_API_KEY`
- `REPORT_FROM_EMAIL`

메일 발송 실패 시 DAY 완료 기록은 유지하며 재발송 대기열에 저장합니다. 관리자 강제 완료는 정답률을 100%로 꾸미지 않고 `미측정`으로 보고합니다.

## 실행 방법

```bash
cd webapp
python -m http.server 8080
```

브라우저에서 `http://localhost:8080`을 엽니다.

## Netlify 배포

프로젝트 루트를 Netlify에 연결하면 `netlify.toml` 설정에 따라 `webapp` 폴더가 배포되고 완료 메일 함수가 활성화됩니다.

## 주요 파일

- `webapp/`: 실행 웹앱
- `data/root_units.json`: 최종 ROOT 학습 단위 360개
- `data/learning_units.json`: ROOT 의미·표제어 뜻·어원·예문
- `data/voca18_schedule.json`: 18DAY 및 72개 블록 배치
- `18DAY_배치표.md`: DAY별 분량과 시간표
- `FINAL_검수보고서.md`: 최종 검수 결과와 제한사항
- `tests/FINAL_TEST_RESULTS.txt`: 실제 자동검사 출력
- `tests/validate_package.py`: 데이터·배치 무결성 검사
- `tests/progress_logic_test.mjs`: 진도 로직 회귀검사
- `tests/mail_function_test.mjs`: 메일 함수 단위검사
- `netlify/functions/send-day-report.mjs`: 완료 보고 메일 함수
- `scripts/`: 원자료 기반 데이터 재생성 스크립트
